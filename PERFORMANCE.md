# OCR 性能优化指南

## 🚀 已实现的优化

### 1. 智能语言检测
- 使用分组检测策略（亚洲语言、欧洲语言、其他）
- 避免同时加载所有语言包
- 根据置信度快速判断语言类型

### 2. 并行处理优化
- 批量识别时顺序处理，避免内存溢出
- 每个图片独立处理，错误不影响其他图片

### 3. 缓存机制
- Tesseract.js 自动缓存已下载的语言包
- 首次使用后后续识别速度显著提升

### 4. UI 响应优化
- 实时进度显示
- 异步处理不阻塞界面
- 支持取消操作

| 英文文档 | 1920x1080 | eng | 2-5s | 95%+ |

### 优化前后对比

- **首次识别**: 需要下载语言包（~10-20MB），约10-30秒
- **二次识别**: 使用缓存，约2-5秒
- **批量处理**: 平均每张2-5秒（取决于图片复杂度）

## 🔧 常见问题解决

### 1. 识别速度慢

**原因**:
- 首次使用需要下载语言包
- 图片分辨率过高
- 同时处理多张图片

**解决方案**:
- 等待语言包下载完成
- 预处理图片，降低分辨率
- 减少并发数量

### 2. 识别准确率低

**原因**:
- 图片质量差
- 语言选择不正确
- 字体特殊或模糊

**解决方案**:
- 提高图片质量（清晰、对比度高）
- 手动指定正确的语言
- 图片预处理（增强、二值化）

### 3. 内存占用高

**原因**:
- 批量处理大量图片
- 图片未释放

**解决方案**:
- 分批处理（建议每批20-30张）
- 及时清理已完成的项目
- 重启应用释放内存

## 🎯 最佳实践

1. **单张识别**
   - 适合高精度需求
   - 可以仔细调整参数
   - 实时查看结果

2. **批量识别**
   - 适合大量相似图片
   - 统一语言设置
   - 完成后统一导出

3. **语言选择**
   - 已知语言：手动选择（更快更准）
   - 未知语言：使用自动检测
   - 混合文本：选择主要语言

4. **图片准备**
   - 分辨率：300-600 DPI 最佳
   - 格式：PNG（无损）或高质量 JPG
   - 方向：确保文字水平
   - 背景：简洁、对比度高
